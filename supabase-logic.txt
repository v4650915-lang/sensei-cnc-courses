# ЛОГИКА АВТОРИЗАЦИИ SUPABASE ДЛЯ TAKUMI CNC

Для реализации закрытого доступа к урокам, Antigravity должен настроить следующую логику в клиенте Supabase:

1. ТАБЛИЦА ПОЛЬЗОВАТЕЛЕЙ (Users):
Используем стандартную аутентификацию Supabase (Email Auth).
Помимо стандартных полей, в таблице (или в public.profiles) нужно поле `access_level` (или `has_access` типа boolean). По умолчанию при регистрации оно равно `false`.

2. ПРОЦЕСС РЕГИСТРАЦИИ:
- Пользователь вводит Email на сайте в форму "Присоединиться / Войти".
- Supabase отправляет Magic Link (ссылку для входа без пароля) или стандартный код подтверждения.
- Пользователь подтверждает почту и попадает в личный кабинет (`dashboard.html`).

3. МЕХАНИКА ДОСТУПА К УРОКАМ:
- Сразу после регистрации ученик видит в кабинете заглушку: "Доступ к материалам закрыт. Ожидайте подтверждения от преподавателя или введите промокод".
- Когда Владимир (админ) в консоли Supabase меняет флаг `has_access` на `true`, у пользователя в кабинете появляются ссылки на `lessons/week1/` и `lessons/week2/`.

4. ЗАЩИТА СТРАНИЦ (Routing Guard):
На всех страницах в папке `/lessons/` должен стоять JS-скрипт проверки сессии:
```javascript
const { data: { session } } = await supabase.auth.getSession()
if (!session) {
    window.location.href = '/index.html'; // редирект на главную, если не вошел
}
// Дополнительно проверяем флаг доступа из базы
